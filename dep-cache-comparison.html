Recommended Strategy
Strategy name Read-only host cache + optional pre-populate Pre-populate node_modules + redirect store for misses Read-only host cache + redirect misses to worktree (hybrid)
Description

Bind-mount ~/.cache/uv read-only into the sandbox. On a full lock-file match, optionally reflink/copy a pre-built .venv/ into the worktree for faster startup.

UV_CACHE_DIR=/cache/uv # read-only bind # optional: cp -r /snapshots/.venv .venv

Pre-populate node_modules/ via reflink when pnpm-lock.yaml matches. Redirect the pnpm store to a writable path so cache misses can still resolve.

# .npmrc store-dir=/writable/pnpm-store # pre-populate on lock match: cp -r --reflink=auto /snapshots/node_modules .

Bind-mount ~/go/pkg/mod read-only. Set GOMODCACHE to a writable per-worktree path so any cache misses are written locally without touching the host cache.

GOMODCACHE="$WORKTREE/.gomodcache" # host ~/go/pkg/mod still consulted # read-only — Go checks it first
